# Internationalization (i18n) Guide

This project ships a JSON-based localization system with support for 19 languages, including RTL languages.

## Locale Files

- Source locale: `locales/en/translations.json`
- Additional locales: `locales/*/translations.json`
- Runtime translation hooks: `lib/hooks/useTypedTranslation.ts` and related i18n helpers under `lib/i18n/`

## Adding a New Translation Key

1. Add the key to `locales/en/translations.json`.
2. Add the same key to every non-English locale file under `locales/`.
3. Keep ICU placeholders consistent across all languages.
4. Run:

```bash
npm run check:i18n
```

## Using Translations in Code

- Use typed translation hooks in UI code.
- Prefer translation keys over hardcoded user-facing strings.
- Keep accessibility labels (`aria-label`, `title`, status text) localized.

## RTL Support

- RTL locales are supported and should be tested for layout, focus order, and text clipping.
- Validate critical pages (`/home`, `/calendar`, `/map`, `/settings`) in at least one RTL locale before merging large UI changes.

## Validation Commands

```bash
npm run check:i18n
npm run check
```
